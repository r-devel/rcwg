    ```

    ```

    ```

    $ :function (x)
    ```

    * people may skip tests on CRAN, so may be best to download actual GitHub repo and run those tests

    * people can use .Rbuildignore to exclude tests from tarball that goes to CRAN so won't be in CRAN mirror.

## CRAN packages that link to other packages

* Largely dominated by Rcpp/sister packages. rstan is also heavily linked.
* Motivation: wanted to link to code in cli package that generates a progress bar.
* Need to have inst/include in package to provide a header file or similar for other packages to use.

    * A lot of Posit packages/packages in r-lib do expose C code, but underused.

    * Potentially collaborate with Posit on blog post with example of how this can be used.

    * Possibly could be documented in Writing R Extensions manual.


## useR! Lightning Talks

* Check if there are any constraints on format, e.g. 1 slide per 30 seconds
* Need to be concise, not much time to give background and take people on a journey. 
* Focus on one big idea: how to get there and how people can use it.
* Watch lightning talks from previous useR! to see what works well.
* Practice a few times, presenting to different people if possible to get feedback.
